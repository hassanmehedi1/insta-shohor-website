1. What is Event-Loop?

Ans: From my understanding an event loop can not be described in just few lines but I'll try to make this fit. So first of all we all know that javascript is a single threaded programming language which means that js can execute only one line code at a time. In js it reads code from top to bottom way. In this way if it finds any function or execution context it send those to the call stack. A call stack works in a LIFO way which means last in first out. So if any function takes big time to execute the whole browser can stuck until the function finishes execution. Thats when event loop arrives, so what it does is when someone calls a setTimeout function, js detects it as asynchronous function and as soon as it gets to call stack, the call stack sends it another place called Web-Api. Here the timer gets set and executes for the setTimeout function after the timer finishes it goes to callBack Queue(callBack queue works in FIFO way), where it waits for Call-Stack to get empty. after call-stack finishes all the synchronous functions/executions and gets empty, an Event-loop throws setTimeout task to the call-stack and it gets Executed. So basically an event-loop helps the console to finish tasks that does not need any time immediately without jamming the browser/console and make the code work in more efficient way.
